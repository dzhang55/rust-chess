Each client thread waits for input (or disconnects) from its respective clients and relays the appropriate messages via the relay MPSC channel.
listen

Create the relay MPSC (multi-producer/single-consumer) channel, spawn the relay thread, then listen for WebSocket clients and spawn their threads.
relay_thread

The relay thread handles all Actions received on its MPSC channel. If it receives an 'Action::Select', it will calculate the potential_moves that can be made by that cell, and then relay an 'Action::Moves' to the sender only. If it receives an 'Action::Move', it will adjust the board state, check for check and checkmate, switch turns, and then send an 'Action::Board' with all the state to all clients. If it receives any other 'Action', it will relay the 'Action' verbatim to all clients.
